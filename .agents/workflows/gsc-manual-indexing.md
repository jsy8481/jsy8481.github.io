이 워크플로우는 브라우저 서브에이전트를 사용하여 구글 서치 콘솔(Google Search Console)에서 특정 URL이나 sitemap.xml의 전체 URL에 대해 '색인 생성 요청'을 자동으로 수행하는 방법입니다.

### ⚠️ 사전 준비
1. 크롬 등 브라우저 창을 열고 구글 계정으로 로그인합니다.
2. [Google Search Console URL 검사 도구](https://search.google.com/search-console)에 접속하여 대상 사이트 속성(예: `jsy8481.github.io`)을 선택해 둡니다.
3. **효율적인 병렬 작업을 위해 위 Search Console 탭을 총 3개 복제해서 열어 둡니다.**
4. 기준이 되는 사이트맵 주소는 `https://jsy8481.github.io/sitemap.xml` 입니다. 이 XML에서 추출한 주소(및 사이트맵 주소 자체)를 사용해 색인을 요청합니다.

### 실행 방법

에이전트에게 **"구글 서치 콘솔 수동 인덱싱 스킬 실행해줘"** 또는 **"/ㅌ₩gsc-manual-indexing 실행해줘"** 라고 요청합니다.
(특정 URL만 하고 싶다면 "이 URL들만 색인 요청해줘"라고 덧붙일 수 있습니다.)

### AI 에이전트 행동 지침 (Agent Instructions)
사용자가 이 워크플로우를 요청하면 다음 순서로 작업을 수행하세요.

1. `curl -s https://jsy8481.github.io/sitemap.xml` 명령이나 파일 읽기를 통해 전체 URL 목록을 추출합니다.
2. 추출한 URL들을 **특정 섹션(카테고리) 단위로 그룹화**합니다. (예: `jotai`, `tanstack-query`, `drizzle`, `nestjs` 등)
3. 사용자에게 추출된 그룹들을 보여주고, **"어느 섹션부터 색인 요청을 진행할까요?"** 하고 물어봅니다.
4. 사용자가 특정 섹션을 선택하면, 해당 섹션의 URL 목록만 작업 대상으로 지정합니다.
5. 사용자의 활성화된 브라우저 탭 목록을 확인하고, 구글 서치 콘솔(Search Console) 탭이 **3개** 열려 있는지 확인합니다. 부족하다면 사용자에게 3개를 열어달라고 요청합니다.
6. `browser_subagent`를 호출하여 아래 지침과 3개의 Page ID를 모두 전달합니다.

**Browser Subagent 지침 예시:**
```text
목표: 제공된 특정 섹션의 URL 목록에 대해 구글 서치 콘솔에서 색인 생성 요청을 수행합니다.
대상 페이지: (현재 열려있는 Search Console 탭 3개의 Page ID: [ID1, ID2, ID3])
대상 URL 목록: [url1, url2, url3, url4, url5...]

작업 방식 (3개 탭을 활용한 분산 처리):
1. URL 목록을 3개씩 묶어서 각각 Tab 1, Tab 2, Tab 3에 할당합니다.
2. Tab 1로 이동하여 첫 번째 URL을 검색창에 입력하고 Enter.
3. 곧바로 Tab 2로 이동하여 두 번째 URL을 검색창에 입력하고 Enter.
4. 곧바로 Tab 3로 이동하여 세 번째 URL을 검색창에 입력하고 Enter.
5. 각 탭에서 "Google 인덱스에서 데이터 가져오는 중..." 로딩이 끝날 때까지 대기합니다 (3개 탭이 동시에 로딩되어 시간을 절약함).
6. 로딩이 끝난 탭부터 차례대로 "색인 생성 요청" 버튼을 클릭합니다.
7. 각 탭에 "색인 생성 요청됨" 성공 메시지가 뜨면 창을 닫고, 다음 3개의 URL 묶음으로 작업을 반복합니다.
8. 할당량 초과(Rate Limit) 등의 에러가 발생하면 오류가 발생한 URL부터 작업을 중단하고 결과를 보고합니다.
```

### 주의 사항
* 단기간에 너무 많은 수동 색인 요청을 보내면 구글 서치 콘솔에서 **일일 할당량 초과(Rate Limit)** 에러가 발생할 수 있습니다. 
* 하루에 최대 10~50개 정도의 요청만 권장합니다.
